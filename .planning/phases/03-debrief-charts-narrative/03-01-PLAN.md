Extend (do NOT replace) `beergame/engine/metrics.py`. Keep the existing `peak_orders` and max-based `bullwhip_ratio` functions intact — Phase 1's `tests/test_bullwhip_emerges.py` depends on them. Add four new public symbols:

1. **`CostRow`** — frozen, slotted dataclass:
   ```python
   @dataclass(frozen=True, slots=True)
   class CostRow:
       role: Role
       holding: float
       backorder: float
       total: float        # invariant: equals s.cost_history[-1] for that role
   ```

2. **`variance_bullwhip_ratio(state: GameState) -> float`** — DEB-03 headline metric:
   - Compute `denom = pvariance(state.customer_demand_history)` and `numer = pvariance(state.stations[Role.FACTORY.value].orders_placed_history)`.
   - Wrap each `pvariance` call in `try/except StatisticsError` and return `0.0` on error (empty / length-1 input — Pitfall 7 in RESEARCH.md).
   - If `denom == 0`, return `0.0` (defensive — canonical step demand has nonzero variance, but a future flat-demand scenario should not crash).
   - Otherwise return `numer / denom`.

3. **`per_echelon_amplification(state: GameState) -> dict[Role, float]`** — DEB-03 per-station ratios:
   - Same denominator (`pvariance` of `customer_demand_history`) for all four roles.
   - For each `Role`, compute `pvariance(s.orders_placed_history)` and divide; on `StatisticsError` or `denom == 0`, the role's value is `0.0`.
   - Return a dict keyed by every `Role` member (length-4) — callers iterate `Role` and look up.

4. **`cost_breakdown(state: GameState) -> tuple[CostRow, ...]`** — DEB-04 per-station decomposition:
   - For each `role in Role`:
     - `s = state.stations[role.value]`
     - `holding = HOLDING_COST * sum(max(0, x) for x in s.inventory_history)`
     - `backorder = BACKORDER_COST * sum(s.backlog_history)`
     - `total = holding + backorder`
   - Mirror `engine/costs.py::weekly_cost` EXACTLY: that function uses `HOLDING_COST * max(0, station.inventory) + BACKORDER_COST * station.backlog` and is called per-tick from `record_state`, so summing the same formula over `inventory_history` and `backlog_history` must reproduce `cost_history[-1]` to floating-point tolerance.
   - Use `HOLDING_COST` and `BACKORDER_COST` imported from `beergame.config.costs` (NOT redefined).
   - Return a tuple of 4 `CostRow` instances, one per Role, in `Role` iteration order (RETAILER, WHOLESALER, DISTRIBUTOR, FACTORY).

Required imports to add at the top of the file (alongside the existing `from beergame.engine.state import GameState, Role`):
```python

Do NOT use NumPy. Do NOT use pandas. Use `statistics.pvariance` (POPULATION variance — we have the complete 36-week series, not a sample). Do NOT change the existing `bullwhip_ratio` signature or semantics (it's max-based and used by Phase 1's calibration test).

Add a one-paragraph module docstring noting that this file now serves both Phase 1 (peak-based bullwhip calibration) AND Phase 3 (variance-based amplification + cost decomposition).

from beergame.ai.base import ConstantOrderAgent
from beergame.ai import ShipmentAnchorAndAdjustAgent
from beergame.engine.state import Role, new_game
from beergame.engine.tick import advance_week, is_game_over

@pytest.fixture(scope="session")
def canonical_done_state():
    """A fully-played canonical 36-week game (seed=42, RETAILER human, Sterman AI on W/D/F).

    Session-scoped: simulating 36 weeks costs ~30ms; sharing the result across
    chart + metric tests avoids redundant work. Phase 1's test_bullwhip_emerges
    already proves this exact seed produces a recognisable bullwhip (max ratio
    in [2.0, 4.0]); Phase 3 tests build on that result.
    """
    state = new_game(player_role=Role.RETAILER, seed=42)
    agents = {r: ShipmentAnchorAndAdjustAgent() for r in Role if r != Role.RETAILER}
    while not is_game_over(state):
        # Retailer (player) plays a naive constant-4 — what we care about
        # for Phase 3 is whatever bullwhip the Sterman AI produces upstream.
        state = advance_week(state=state, player_order=4, ai_agents=agents)
    return state
```
