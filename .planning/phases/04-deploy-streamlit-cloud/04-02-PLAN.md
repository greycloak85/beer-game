```bash
DEPLOY-03, DEPLOY-04, DEPLOY-05, DEPLOY-06.
EOF
)"
fi

# 2. Create the public repo on GitHub and push master in one shot.
#    --source . sets the upstream to the current dir.
#    --remote origin wires the remote.
#    --push pushes the current branch (master) immediately.
#    --description gives the GitHub page a one-liner.
#    --public is the non-negotiable bit (DEPLOY-01).
gh repo create greycloak85/beer-game \
  --public \
  --source . \
  --remote origin \
  --push \
  --description "Single-player web simulation of the MIT Beer Distribution Game (Sterman 1989). Play one of four supply-chain stations against AI opponents and see the bullwhip effect emerge in the post-game debrief."

# Expected: prints something like
#   ✓ Created repository greycloak85/beer-game on GitHub
#   ✓ Added remote https://github.com/greycloak85/beer-game.git
#   ✓ Pushed commits to https://github.com/greycloak85/beer-game.git
```

**Why this exact invocation:**
- `--public` — DEPLOY-01 requires public; Streamlit Community Cloud free tier ONLY connects to public repos.
- `--source .` + `--remote origin` + `--push` together do the work of `git init` (already done — this repo has commits) + `git remote add origin` + `git push -u origin master` in one step, which is the documented `gh` idiom and the orchestrator's locked decision.
- The description is short, written for a curious GitHub visitor — mirrors the README's tagline.
- We push `master` (the project's branch name per the orchestrator's environment notes). Do NOT rename to `main` — that's a deferred decision and the user hasn't asked.

**If `gh repo create` fails:**
- `--source` requires a clean-enough state. If gh complains about an unclean working tree, the pre-push check missed something — investigate, do NOT force.
- If gh hits an auth wall (token scope missing `repo`), `gh auth refresh -s repo` is the fix — but only run it after surfacing the issue to the user via SUMMARY; don't refresh tokens silently.
- If the network drops mid-push, retry with `git push -u origin master` (the remote was already added). Do not rerun `gh repo create` — that errors on already-exists.
