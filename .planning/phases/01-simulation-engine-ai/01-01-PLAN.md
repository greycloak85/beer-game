    **CRITICAL design clarification — Factory's two distinct lead-time channels (see also v1 limitation note on customer_demand timing in Task 2):**

    Factory has TWO separate flows that are easy to conflate:

    1. **Inbound order channel (Distributor → Factory):** Distributor places an order, that order travels via the standard 1-week mailing delay just like every other inter-echelon order. This is `Factory.incoming_orders`, length 1, pre-loaded with `(4,)` at equilibrium. THIS IS WHAT `ORDER_PIPELINE_LEN_FACTORY = 1` CONTROLS.

    2. **Factory's own production-start path (Factory's placed order → Factory's incoming_shipments):** When Factory itself decides to order N this week, it does NOT go to any upstream order queue (there is no upstream — Factory is the source of the chain). Instead, N becomes a production start: it goes directly into Factory's own `incoming_shipments` pending slot, where it sits for `SHIPPING_PIPELINE_LEN = 2` weeks of production lead time before being available as Factory inventory.

    The phrase "no order delay at Factory" from the Sterman/JASSS literature refers to flow #2 (Factory's own order→production path skips the mailing step), NOT flow #1 (the Distributor→Factory order channel still uses the canonical 1-week mailing delay like every other inter-echelon channel).

    Plan 01 v1 had `ORDER_PIPELINE_LEN_FACTORY = 0`, which incorrectly applied "no order delay" to flow #1 — that caused Distributor's orders TO Factory to be silently discarded in `advance_pipelines` (the `else: new_orders = ()` branch). The fix is `ORDER_PIPELINE_LEN_FACTORY = 1` and pre-loading `incoming_orders=(4,)` on the Factory at game start.

    **1. `beergame/engine/tick.py`** — implement EXACTLY per 01-RESEARCH.md "Tick Implementation" section.

    Structure: five named pure functions, each `(GameState) -> GameState` or `(GameState, ...) -> GameState`, composed in literal source order by `advance_week`. The five steps and the data flow MUST follow the research's "Pipeline data flow — DECISION":
    - Transient fields `_pending_in_shipment` and `_pending_in_order` on `StationState` carry the data set by step 2 (fill_orders) and step 4 (place_orders) into step 5 (advance_pipelines), which consumes them and shifts the queues.
    - Use the additional transient fields `_demand_to_fill` (set in step 2, consumed by step 3 to append to `orders_received_history`) and `_shipped_this_tick` (set in step 2, consumed by step 3 to append to `shipments_sent_history`).

    Step-by-step:

    ```python
    """Canonical Beer Game tick — five named steps in fixed order.

    SEQUENCE (Sterman 1989, JASSS 17(4):2 §3): receive shipments → fill orders →
    record state → place new orders → advance pipelines. Any rearrangement breaks
    the bullwhip. The order is asserted by tests/test_tick_invariants.py.

    FACTORY ASYMMETRY (critical, see also config/scenarios.py docstring):
    - Factory's INBOUND order channel (orders FROM Distributor) uses the SAME 1-week
      mailing delay as every other inter-echelon order channel — `ORDER_PIPELINE_LEN_FACTORY = 1`,
      `Factory.incoming_orders` is length 1, pre-loaded with `(4,)` at equilibrium.
    - Factory's OWN order→production path skips the mailing channel: when Factory
      decides to order N this week, N goes directly into Factory's own
      `_pending_in_shipment` (consumed in step 5 into its own `incoming_shipments`
      back slot), where it sits for `SHIPPING_PIPELINE_LEN = 2` weeks of production.
    """
    from dataclasses import replace
    from typing import Callable
    from beergame.engine.state import (
        GameState, StationState, Role, StationView, RetailerView,
        new_game, build_station_view,
    )
    from beergame.engine.demand import demand_for_week
    from beergame.engine.costs import weekly_cost
    from beergame.ai.base import Agent
    from beergame.config.scenarios import EQUILIBRIUM_THROUGHPUT


    def _idx(role: Role) -> int:
        return role.value


    def receive_shipments(state: GameState) -> GameState:
        """Step 1: add slot-0 of each station's incoming_shipments to inventory;
        zero that slot (step 5 will shift it out)."""
        new_stations = []
        for s in state.stations:
            arriving = s.incoming_shipments[0] if s.incoming_shipments else 0
            new_shipments = (0,) + s.incoming_shipments[1:] if s.incoming_shipments else ()
            new_stations.append(replace(
                s, inventory=s.inventory + arriving,
                incoming_shipments=new_shipments,
            ))
        return replace(state, stations=tuple(new_stations))


    def fill_orders(state: GameState, demand_fn: Callable[[int, int], int] = demand_for_week) -> GameState:
        """Step 2: receive incoming order (Retailer: customer demand; others: incoming_orders[0]),
        ship min(inventory, demand+backlog) downstream, stash transient fields for step 3 + step 5.

        Note: Factory's incoming_orders is length 1 (canonical mailing delay from Distributor).
        Reading slot 0 gives the order Distributor placed one week ago — identical pattern to W/D.
        """
        next_week = state.week + 1
        customer_demand_this_week = demand_fn(next_week, state.total_weeks)
        new_customer_history = state.customer_demand_history + (customer_demand_this_week,)

        # Pass 1: determine demand each station must fill.
        demands = [0, 0, 0, 0]
        for role in Role:
            i = _idx(role)
            s = state.stations[i]
            if role == Role.RETAILER:
                demands[i] = customer_demand_this_week
            else:
                # Non-Retailer (including Factory): order arrives at slot 0 of incoming_orders.
                demands[i] = s.incoming_orders[0] if s.incoming_orders else 0

        # Pass 2: compute fills, stash transient outgoing shipment on RECEIVER's _pending_in_shipment.
        new_stations = list(state.stations)
        # Pre-compute shipped[] so we can route to downstream's pending field afterward.
        shipped = [0, 0, 0, 0]
        post_fill_inv = [0, 0, 0, 0]
        post_fill_bl = [0, 0, 0, 0]
        for role in Role:
            i = _idx(role)
            s = new_stations[i]
            total_need = demands[i] + s.backlog
            shipped[i] = min(s.inventory, total_need)
            post_fill_inv[i] = s.inventory - shipped[i]
            post_fill_bl[i] = total_need - shipped[i]

        # Pass 3: apply (inventory, backlog, transient _demand_to_fill + _shipped_this_tick),
        # and route the shipment to DOWNSTREAM's _pending_in_shipment (Retailer ships to
        # external customer — that shipment leaves the system, nothing to enqueue).
        for role in Role:
            i = _idx(role)
            s = new_stations[i]
            new_stations[i] = replace(
                s,
                inventory=post_fill_inv[i],
                backlog=post_fill_bl[i],
                _demand_to_fill=demands[i],
                _shipped_this_tick=shipped[i],
            )

        for role in Role:
            i = _idx(role)
            if role == Role.RETAILER:
                continue  # ships to external customer — leaves the system
            downstream_idx = i - 1
            ds = new_stations[downstream_idx]
            new_stations[downstream_idx] = replace(
                ds, _pending_in_shipment=ds._pending_in_shipment + shipped[i],
            )

        return replace(state, stations=tuple(new_stations),
                       customer_demand_history=new_customer_history)


    def record_state(state: GameState) -> GameState:
        """Step 3: append post-fill inventory/backlog/orders_received/shipments_sent + cumulative cost."""
        new_stations = []
        for s in state.stations:
            w_cost = weekly_cost(s)
            prev_cum = s.cost_history[-1] if s.cost_history else 0.0
            new_stations.append(replace(
                s,
                inventory_history=s.inventory_history + (s.inventory,),
                backlog_history=s.backlog_history + (s.backlog,),
                orders_received_history=s.orders_received_history + (s._demand_to_fill,),
                shipments_sent_history=s.shipments_sent_history + (s._shipped_this_tick,),
                cost_history=s.cost_history + (prev_cum + w_cost,),
                _demand_to_fill=0,
                _shipped_this_tick=0,
            ))
        return replace(state, stations=tuple(new_stations))


    def place_orders(state: GameState, player_order: int,
                     ai_agents: dict[Role, Agent]) -> GameState:
        """Step 4: each station decides an order. Routes to upstream's _pending_in_order
        for R/W/D, or to Factory's own _pending_in_shipment for the Factory.

        FACTORY ASYMMETRY: Factory's placed order (its own production start) goes to
        Factory's own _pending_in_shipment — it does NOT go into any upstream order
        channel (there is no upstream — Factory is the source). The Distributor→Factory
        order channel is separate and handled by the R/W/D branch below (Distributor
        routes its order to Factory's _pending_in_order, which then sits in Factory's
        length-1 incoming_orders pipeline for one tick before being filled).
        """
        orders_placed = [0, 0, 0, 0]
        for role in Role:
            i = _idx(role)
            if role == state.player_role:
                orders_placed[i] = max(0, int(player_order))
            else:
                view = build_station_view(state, role)
                orders_placed[i] = max(0, int(ai_agents[role].decide_order(view)))

        new_stations = list(state.stations)
        # Append placed order to each station's history.
        for role in Role:
            i = _idx(role)
            new_stations[i] = replace(
                new_stations[i],
                orders_placed_history=new_stations[i].orders_placed_history + (orders_placed[i],),
            )
        # Route the orders.
        for role in Role:
            i = _idx(role)
            o = orders_placed[i]
            if role == Role.FACTORY:
                # Factory's OWN order = production start → its own incoming_shipments back slot.
                fs = new_stations[i]
                new_stations[i] = replace(fs, _pending_in_shipment=fs._pending_in_shipment + o)
            else:
                # R/W/D: order goes to upstream's incoming_orders pending slot.
                # For Distributor (i=2), upstream is Factory (i=3) — Factory's incoming_orders
                # is now length 1, so this order will be received by Factory in one tick.
                up_idx = i + 1
                us = new_stations[up_idx]
                new_stations[up_idx] = replace(us, _pending_in_order=us._pending_in_order + o)
        return replace(state, stations=tuple(new_stations))


    def advance_pipelines(state: GameState) -> GameState:
        """Step 5: shift pipelines forward; pending fields become the new back-slot entry."""
        new_stations = []
        for s in state.stations:
            # Shipments: drop slot 0 (already received in step 1), shift forward, append pending.
            if s.incoming_shipments:
                new_shipments = s.incoming_shipments[1:] + (s._pending_in_shipment,)
            else:
                new_shipments = ()
            # Orders: same pattern. Factory's incoming_orders is now length 1 — same as R/W/D.
            if s.incoming_orders:
                new_orders = s.incoming_orders[1:] + (s._pending_in_order,)
            else:
                new_orders = ()
            new_stations.append(replace(
                s,
                incoming_shipments=new_shipments,
                incoming_orders=new_orders,
                _pending_in_shipment=0,
                _pending_in_order=0,
            ))
        next_week = state.week + 1
        new_phase = "done" if next_week >= state.total_weeks else "playing"
        return replace(state, week=next_week, stations=tuple(new_stations), phase=new_phase)


    def advance_week(state: GameState, player_order: int,
                     ai_agents: dict[Role, Agent],
                     demand_fn: Callable[[int, int], int] = demand_for_week) -> GameState:
        """Single canonical tick. Returns a NEW GameState.

        SEQUENCE: receive shipments → fill orders → record state → place new orders → advance pipelines.
        """
        assert state.phase == "playing", f"Cannot advance: phase={state.phase}"
        s1 = receive_shipments(state)
        s2 = fill_orders(s1, demand_fn=demand_fn)
        s3 = record_state(s2)
        s4 = place_orders(s3, player_order, ai_agents)
        s5 = advance_pipelines(s4)
        return s5


    def is_game_over(state: GameState) -> bool:
        return state.phase == "done"


    def simulate_full_game(seed: int, player_role: Role,
                           agents: dict[Role, Agent],
                           demand_fn: Callable[[int, int], int] = demand_for_week,
                           total_weeks: int | None = None) -> GameState:
        """Convenience driver: run all 36 ticks. `agents` must include the player_role
        (the "player" is treated as just another Agent here).

        v1 limitation: build_station_view at loop top uses last week's recorded demand;
        for week 0, falls back to CLASSIC_PRE_STEP_DEMAND. Both supplied demand_fns
        (demand_for_week, constant_demand) start at 4 so this is consistent for v1.
        A structural fix would require threading demand_fn through build_station_view
        and is deferred to v2.

        Used by GATE 1 and GATE 2 tests in Plan 03 — implemented now so Plan 03 only
        needs to write the two test files.
        """
        kwargs = {"player_role": player_role, "seed": seed}
        if total_weeks is not None:
            kwargs["total_weeks"] = total_weeks
        state = new_game(**kwargs)
        while state.phase == "playing":
            # v1 limitation: build_station_view at loop top uses last week's recorded
            # demand; for week 0, falls back to CLASSIC_PRE_STEP_DEMAND. Both supplied
            # demand_fns start at 4 so this is consistent.
            player_view = build_station_view(state, player_role)
            player_order = agents[player_role].decide_order(player_view)
            ai_subset = {r: a for r, a in agents.items() if r != player_role}
            state = advance_week(state, player_order, ai_subset, demand_fn=demand_fn)
        return state
    ```

    **Tick 1 walkthrough — REVISED design (Factory has canonical 1-week inbound order delay, every station plays `ConstantOrderAgent(4)`, constant customer demand=4):**

    Initial state (week 0):
    - Every station: inventory=12, backlog=0, incoming_shipments=(4, 4), incoming_orders=(4,) — INCLUDING Factory.
    - customer_demand_history = ()

    Step 1 (receive_shipments): each station receives the slot-0 shipment of 4. Inventory becomes 16 at every station. incoming_shipments[0] zeroed to (0, 4).

    Step 2 (fill_orders): customer_demand_this_week=4 (constant). demands=[4, 4, 4, 4] (Retailer faces customer demand=4; W/D/Factory each face the order from incoming_orders[0]=4). Each station has total_need = 4+0 = 4, ships min(16, 4)=4, post_fill_inv=12, post_fill_bl=0. Retailer's ship-of-4 leaves system (no enqueue). Wholesaler ships 4 to Retailer (_pending_in_shipment+=4 on Retailer). Distributor ships 4 to Wholesaler. Factory ships 4 to Distributor.

    Step 3 (record_state): every station appends inventory=12, backlog=0 to history. Cost = 0.5*12 = 6.0 per station.

    Step 4 (place_orders): every station's ConstantOrderAgent(4) returns 4. orders_placed=[4,4,4,4]. Routes: Retailer's 4 → Wholesaler._pending_in_order=4. Wholesaler's 4 → Distributor._pending_in_order=4. Distributor's 4 → Factory._pending_in_order=4. Factory's 4 → Factory._pending_in_shipment (its own production start), so Factory._pending_in_shipment was already 0 (no upstream shipment was routed to Factory in step 2 because Factory has no upstream supplier in the chain) and now becomes 4.

    Wait — re-check Factory's _pending_in_shipment. In step 2, Factory shipped 4 to Distributor (downstream), so Distributor._pending_in_shipment+=4. Factory itself receives nothing from "upstream" in step 2 because there is no upstream (Factory is the source). So Factory._pending_in_shipment ends step 2 at 0. Step 4 adds 4 (its own production start). End of step 4: Factory._pending_in_shipment=4.

    Step 5 (advance_pipelines): for each station, incoming_shipments=(0,4) becomes (4, _pending_in_shipment). For Retailer, Wholesaler, Distributor: _pending_in_shipment=4 (received from downstream supplier), so incoming_shipments=(4, 4). For Factory: _pending_in_shipment=4 (its own production start), so Factory.incoming_shipments=(4, 4) — IDENTICAL pattern. incoming_orders=(4,) becomes (4,) again (slot[1:] is empty + _pending_in_order=4 for everyone; for Factory the Distributor's order of 4 is now sitting in Factory.incoming_orders=(4,)). week becomes 1.

    Result after tick 1: every station has inventory=12, backlog=0, incoming_shipments=(4,4), incoming_orders=(4,) — IDENTICAL to the initial state. The system is in perfect equilibrium and stays there forever under ConstantOrderAgent(4) + constant demand=4. **Factory inventory stays at 12**, NOT 16 as in the old broken design. This is what GATE 1 will assert.
